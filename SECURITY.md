# Security

Do not report real LeetCode session cookies, CSRF tokens, source code, or
submission data in public issues.

Credentials must only be supplied through documented environment variables or
an operating-system credential provider. They must never be committed, logged,
included in fixtures, or passed as tool-call arguments.

Report suspected vulnerabilities through
[GitHub Private Vulnerability Reporting](https://github.com/Li7777777/pi-leetcode-tools/security/advisories/new).

Do not open a public issue, discussion, or pull request for a suspected
vulnerability. Remove all LeetCode credentials, source code, submission data,
account identifiers, and response bodies from the report unless the maintainers
provide an appropriate private transfer method.
