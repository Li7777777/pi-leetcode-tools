# Security

Do not report real LeetCode session cookies, CSRF tokens, source code, or
submission data in public issues.

Credentials must only be supplied through documented environment variables or
an operating-system credential provider. They must never be committed, logged,
included in fixtures, or passed as tool-call arguments.

Until a private disclosure channel is published, open a minimal public issue
that contains no secret or exploit detail and request a private contact route.
